## 🌿 Convención de ramas

Para mantener un flujo de trabajo ordenado, se define la siguiente convención de nombres para las ramas:

### 🔹 Tipos de ramas

* **feature/** → nuevas funcionalidades
* **fix/** → corrección de errores

### 🔹 Formato

```bash
feature/nombre-tarea
fix/nombre-bug
```

### 🔹 Ejemplos

```bash
feature/login-T01
feature/registro-T06
fix/error-validacion-email
fix/bug-login
```

### 🔹 Reglas

* Usar minúsculas
* Separar palabras con guiones (`-`)
* Ser descriptivo pero conciso

> Todos los cambios deben realizarse mediante **Pull Requests** hacia la rama `main`.
